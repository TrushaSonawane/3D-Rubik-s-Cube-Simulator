Materials (colors) are assigned based on their initial position to form the classic solved state.Raycasting for Interaction: A THREE.Raycaster is used to detect which face of which cubie the user has clicked/tapped on.Drag-to-Twist Logic: The onPointerDown, onPointerMove, and onPointerUp handlers track the drag direction (horizontal/vertical swipe) and the clicked face's normal to mathematically determine the correct axis ('x', 'y', or 'z') and direction of the intended 90-degree layer rotation.Pivot Rotation: To animate a layer twist, the relevant 9 cubies are temporarily attached to a single THREE.Object3D (a pivot). This pivot is rotated $90^\circ$ around the required axis.State Update: Once the animation is complete, the cubies are detached from the pivot, and their final rotation and position are snapped to the grid (using cubie.applyMatrix4(pivot.matrix)) to maintain state integrity for the next move.üöÄ Getting StartedPrerequisitesYou only need a modern web browser to run the simulator.Installation & Running LocallyClone the repository:Bashgit clone [YOUR-REPO-LINK]
